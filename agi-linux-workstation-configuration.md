# Configuração de Workstation Linux para Projeto AGI (até R\$ 10.000)

## Introdução

Desenvolver um projeto de **AGI (Artificial General Intelligence)** focado em decomposição de perguntas complexas, consulta a bases científicas e uso de memória *RAG* exige uma workstation robusta. Essa máquina precisa suportar **múltiplos processos Node.js em paralelo**, um **banco de dados SQL local** (por exemplo, PostgreSQL) com alto desempenho, além de manter vários editores e ferramentas de desenvolvimento abertos simultaneamente. Neste relatório, discutimos os **componentes cruciais** para evitar gargalos, apontamos onde é possível economizar sem perda significativa de performance e apresentamos **três configurações sugeridas** (duas montagens desktop e uma workstation pronta), com preços estimados e links para lojas confiáveis no Brasil.

## Componentes Cruciais para Desempenho

### CPU (Processador) – Muitos Núcleos e Desempenho Paralelo

O processador é o **cérebro** da workstation. Para executar várias instâncias Node.js em paralelo e processar consultas de banco de dados complexas, é fundamental ter **múltiplos núcleos e threads disponíveis**. Node.js em si utiliza um único *thread* por processo, mas pode aproveitar *clusters* de processos para escalar em máquinas multi-core. Assim, vários núcleos permitirão rodar instâncias paralelas sem competir por CPU, melhorando a capacidade de atender requisições simultâneas.

Empresas recomendam CPUs com **alto número de núcleos, hyperthreading e clock elevado** para cargas de trabalho intensivas. No contexto do PostgreSQL, por exemplo, consultas podem ser executadas em paralelo por diferentes núcleos, beneficiando-se de um processador moderno com muitos cores. Em suma, **quanto mais núcleos e threads, melhor** o desempenho multitarefa e a escalabilidade do sistema. Uma CPU topo de linha como o AMD Ryzen 9 ou Intel Core i7/i9 vai evitar gargalos de processamento.

**Resumo:** Priorize CPUs com **>= 12 núcleos (24 threads)**. Isso garante que os diversos processos (Node.js, banco de dados, indexadores, etc.) rodem simultaneamente sem estrangular a máquina. Exemplos: AMD Ryzen 9 7900X/7950X ou Intel Core i7/i9 de 13ª geração. Essas CPUs fornecem potência de sobra para “mover rapidamente grandes quantidades de dados” em fluxos de trabalho pesados.

### Memória RAM – Quantidade Abundante para Multitarefa e Cache

A memória RAM é outro pilar crítico. Com múltiplos serviços rodando (várias instâncias Node, o banco SQL, serviços de busca em texto, ambiente de desenvolvimento, etc.), a workstation deve ter RAM suficiente para evitar *swap* em disco. Além disso, o uso de *RAG* (memória de recuperação aumentada) e caches de banco de dados se beneficia de grande quantidade de memória volátil.

Para cargas de trabalho de IA e servidores de dados, recomenda-se **no mínimo 32 GB de RAM**. Esse montante permite manter em memória caches de consultas frequentes do PostgreSQL e conjuntos de dados científicos utilizados pelo AGI, reduzindo o acesso a disco e acelerando leituras. Conforme destacado em guia de hardware, “Recomendamos pelo menos 32 GB” para aplicações intensivas, já que quantidades maiores de RAM permitem preparar e manipular conjuntos de dados maiores em memória. No caso do PostgreSQL, **mais RAM = mais dados em cache**, o que diminui I/O em disco e agiliza operações de leitura.

**Resumo:** Inclua **32 GB de RAM DDR4 ou DDR5 (preferencialmente 2×16 GB em dual-channel)**. Se o orçamento permitir, 64 GB darão folga adicional para futuros crescimentos. Essa memória abundante garantirá que mesmo múltiplos processos e consultas intensivas não esgotem os recursos, mantendo o sistema responsivo.

### Armazenamento SSD (NVMe) – Altíssimo I/O para Banco de Dados

Para um banco de dados local (como PostgreSQL) e acesso rápido a documentos científicos, o uso de um **SSD NVMe** é essencial. Diferente de um SSD SATA comum (limitado a \~550 MB/s), unidades NVMe via PCIe oferecem **largura de banda muito superior** e latências bem menores. Em cenários de muitos acessos concorrentes (consultas, indexações, leitura de arquivos de conhecimento), o NVMe evita que o armazenamento se torne um gargalo.

Conforme documentação de hardware, **unidades NVMe de alto desempenho** conseguem lidar com ordens de magnitude mais operações por segundo e throughput muito maior, enquanto **HDDs ou mesmo SSDs SATA mais lentos devem ficar restritos a armazenamento de longo prazo**. Ou seja, use o NVMe principal para o sistema, banco de dados e arquivos ativos do projeto, e se precisar de muita capacidade para backup ou dados raramente acessados, um HDD secundário pode complementar.

Na prática, um **SSD NVMe M.2 de 1 TB** é recomendado – esse tamanho equilibra custo e espaço, comportando o SO Linux, os dados do banco, índices e cache de documentos científicos. Por exemplo, um SSD NVMe PCIe 4.0 de 1 TB alcança facilmente 3.000+ MB/s, enquanto o SATA ficaria em \~500 MB/s; essa diferença se traduz em carregamento mais rápido do banco e menor tempo de espera em consultas pesadas. *Benchmarks* indicam que NVMe pode atender **múltiplas requisições simultâneas** com muito mais eficiência, beneficiando aplicações de *data analytics* e bancos de dados intensivos.

**Resumo:** Opte por **SSD NVMe (PCIe 3.0/4.0)** como unidade principal. Um modelo de **1 TB NVMe** (ex: Kingston NV2, WD Blue SN570, Samsung 970/980) oferece ótimo desempenho e custa cerca de R\$ 500\~600, cabendo no orçamento. Isso garantirá que operações do PostgreSQL e leitura/escrita de arquivos grandes ocorram sem gargalos de I/O.

### Placa-mãe, Expansão e Compatibilidade

A placa-mãe deve ser compatível com o processador escolhido e oferecer estabilidade no Linux. Para AMD Ryzen 7000, por exemplo, placas com chipset **B650 ou X670** (socket AM5) suportam esses CPUs e memória DDR5. Já para Intel 13ª gen, placas **B760 ou Z790** (socket LGA1700) seriam indicadas.

Componentes importantes na placa-mãe:

* **VRM robusto:** garante entrega de energia estável à CPU multi-core de alto TDP, evitando *throttling*.
* **Slots de RAM suficientes:** permita pelo menos 4 slots (facilitando expansão futura para 64 GB ou mais).
* **Slots M.2 NVMe:** idealmente 2 ou mais. Assim é possível adicionar outro SSD NVMe no futuro (por exemplo, separar um SSD só para o banco de dados, melhorando I/O).
* **Portas e conexões:** USB abundantes, rede Gigabit (ou 2.5GbE) onboard. Gráficos integrados dispensam placa de vídeo dedicada, mas a placa deve ter saídas de vídeo (HDMI/DP) para usar o vídeo integrado do CPU.
* **Compatibilidade Linux:** chipsets AMD e Intel recentes têm bom suporte. Evite placas Wi-Fi de fabricantes menos suportados (ex.: alguns módulos Wi-Fi 6 podem requerer firmware proprietário – se for essencial, prefira Intel Wi-Fi ou use adaptador separado).

Não é necessário um modelo topo de linha voltado a *overclock* (que são caros). Uma placa **intermediária** é suficiente, contanto que entregue os recursos acima. Por exemplo, a **ASUS TUF B650M-PLUS** (AM5) oferece ótima durabilidade e suporte DDR5, custando cerca de **R\$ 1.430**. Placas B650 tendem a oferecer melhor custo-benefício que X670 nesse caso, pois o projeto não exige recursos extras como multi-GPU.

### GPU (Vídeo) – Integrada vs Dedicada

Para este projeto focado em processamento CPU/IO, a placa de vídeo não é um foco de desempenho. **Não será necessário rodar modelos de IA localmente** ou gráficos 3D pesados, então podemos **economizar utilizando vídeo integrado** do processador ou uma GPU básica.

* **AMD Ryzen 7000**: todos os modelos dessa série vêm com um vídeo integrado simples (iGPU Radeon) suficiente para saída de vídeo no desktop Linux. Isso permite **dispensar GPU dedicada** e economizar alguns milhares de reais.
* **Intel Core**: a maioria dos Core i7/i9 possui iGPU (UHD 770) adequada para uso de monitores e aceleração básica de vídeo. Novamente, sem necessidade de placa dedicada.

Caso se deseje usar múltiplos monitores de alta resolução ou alguma aceleração específica, pode-se considerar uma GPU dedicada **entry-level**. Um exemplo é a **AMD Radeon RX 6400 4GB**, que é uma placa moderna, silenciosa (sem conector de energia externo) e custa por volta de R\$ 800-1.000. Porém, essa é opcional – *tanto o Build 1 quanto o Build 2 abaixo utilizam o gráfico integrado do CPU sem problemas*. Para o Build 3 (workstation Dell), inclui-se uma GPU profissional básica NVIDIA T1000, adequada para uso de múltiplos monitores e compatível com Linux (drivers proprietários da NVIDIA).

### Fonte de alimentação (PSU) e Resfriamento – Estabilidade e Ruído

Embora não diretamente ligados a performance bruta, a fonte e o sistema de resfriamento são vitais para manter o desempenho **consistente e silencioso**:

* **Fonte:** Escolha uma PSU de boa qualidade, certificação **80 Plus Gold (ou superior)**, com potência folgada (650W a 750W é suficiente para as configurações propostas). Fontes de marcas confiáveis (Corsair, Seasonic, EVGA, etc.) fornecem tensão estável, proteções contra surtos e normalmente operam com menos ruído em cargas altas. Uma fonte superdimensionada trabalhará em carga parcial, produzindo menos calor e permitindo que a ventoinha gire mais devagar (menos ruído).
* **Cooler da CPU:** Processadores de alto desempenho podem esquentar quando todos os núcleos estão ativos. Para manter a CPU em boost máximo sem *throttling*, use um bom cooler. No caso do **Ryzen 9 7900 (65W)** do Build 2, o cooler Wraith Prism de fábrica já atende razoavelmente (se desejar mais silêncio, um **air cooler torre dual-fan** ou **water cooler 240mm** reduzirá temperaturas). Já o **Ryzen 9 7950X (170W)** do Build 1 e o **Core i7-13700 (Turbo \~180W)** do Build 3 se beneficiam de coolers mais robustos – recomendam-se modelos como **Noctua NH-D15, DeepCool AK620** (a ar) ou **Water Coolers 240/280mm** de marcas conhecidas. Isso garantirá operação sustentada sem estrangulamento, mesmo sob carga total de todos os núcleos.
* **Gabinete:** Não é preciso um gabinete enorme, mas verifique que tenha **boa ventilação** (frontal e superior) e suporte aos componentes escolhidos (espaço para placas ATX, coolers altos ou radiador, etc). Gabinetes *mid-tower* com 2 a 3 ventoinhas de 120mm fornecem fluxo de ar adequado. Se o ruído for uma preocupação, modelos com tratamento acústico ou fans maiores (140mm) ajudam a diminuir o som. Dito isso, **não há restrições rígidas de tamanho ou ruído** neste projeto, então priorize a eficiência térmica sobre o silêncio, se necessário.

## Onde Economizar sem comprometer a Performance

Montar uma máquina equilibrada envolve direcionar o orçamento para o que traz mais ganho e economizar em componentes que não afetarão o desempenho do projeto. Algumas dicas de otimização de custos:

* **Placa de vídeo dedicada:** Como discutido, pode ser omitida se usarmos os gráficos integrados dos CPUs modernos. Isso economiza uma grande parcela do orçamento (R\$ 800 a R\$ 1500 ou mais). Somente adicione uma GPU discreta se realmente precisar de aceleração gráfica específica ou suporte a mais monitores do que as saídas integradas oferecem.
* **Modelo da CPU:** Dentro de uma mesma geração, o topo de linha costuma ter custo x benefício menor. Por exemplo, o Ryzen 9 7950X (16 núcleos) custa \~R\$3.699, enquanto o Ryzen 9 7900 (12 núcleos) sai por \~R\$2.649 e ainda inclui cooler. Se 12 cores já atendem com folga (o que é provável neste caso, dado que não rodaremos IA pesada localmente), podemos poupar \~R\$1.000+ investindo no 7900 em vez do 7950X. Similarmente, na linha Intel, um Core i7 oferece quase o desempenho de um i9 para essa carga, por bem menos custo. Ajuste a escolha do CPU considerando o orçamento vs. necessidade de núcleos extra.
* **Placa-mãe de linha média:** Evite modelos entusiastas caros (como chipsets X670E ou Z790 de luxo) a não ser que precise de recursos específicos. Uma **B650 (AMD)** ou **B760 (Intel)** de boa marca já oferece tudo necessário. Esses modelos médios economizam algumas centenas de reais que podem ser investidos em mais RAM ou SSD.
* **Clock de RAM e RGB:** Memórias DDR5 de altíssima frequência (6000+ MHz) ou com iluminação RGB são bem mais caras, mas trazem ganho mínimo em aplicações de servidor/CPU. Pode-se optar por módulos **DDR5 5200MHz ou 5600MHz**, que costumam ser mais em conta, sem impacto perceptível no desempenho do Node.js ou do PostgreSQL. Por exemplo, um kit **32GB DDR5 5200MHz** Corsair está em promoção por **R\$ 669,99**, enquanto kits 6000MHz costumam passar de R\$ 1000 – a diferença de performance não justifica o extra para nosso caso.
* **Armazenamento adicional:** A não ser que você tenha um volume enorme de dados científicos localmente, 1 TB NVMe deve bastar inicialmente. Caso precise de mais espaço, uma alternativa barata é adicionar um **HDD SATA de 2 TB** apenas para arquivamento/backup, mantendo os dados ativos no NVMe. HDDs custam \~R\$ 300 para 2 TB, sendo econômicos para armazenamento frio. Isso evita gastar muito em um NVMe de 2 TB agora (que custaria próximo de R\$ 1000).

Em resumo, gaste mais onde há impacto direto no desempenho (CPU, quantidade de RAM, um bom SSD) e economize em extras supérfluos (RGB, componentes top de linha com pequenas melhorias, capacidade exagerada que não será usada imediatamente). Com isso, é possível maximizar a performance dentro do orçamento de R\$10.000.

## Builds Sugeridos (Comparação)

A seguir, apresentamos três configurações que atendem aos requisitos, explorando diferentes abordagens: dois desktops customizados (um priorizando máximo desempenho, outro o melhor custo-benefício) e uma **workstation de marca pronta**. Todos os builds visam rodar Linux com plena compatibilidade e manter o orçamento em até R\$10.000. Incluímos links de referência para componentes em lojas confiáveis do Brasil e um resumo comparativo:

### Build 1 – **Desktop AMD High-End (Desempenho Máximo)**

**Descrição:** Workstation montada com a **máxima potência de CPU dentro do orçamento**, visando longevidade e desempenho sobrando. Indicado se você quer o melhor hardware para multitarefa pesada.

* **CPU:** AMD Ryzen 9 7950X (16 núcleos / 32 threads, \~5.7GHz turbo) – Processador topo de linha AM5, excelente para cargas paralelas. *Preço:* \~R\$ 3.700.
* **Cooler CPU:** Water Cooler 240mm (ex: Cooler Master ML240) ou Air Cooler de alto desempenho (ex: Noctua D15). *Preço:* \~R\$ 400–600. Garante temperaturas baixas no 7950X em carga total.
* **Placa-mãe:** ASUS TUF Gaming B650-PLUS (AM5, DDR5, ATX) – Chipset B650 com robustez e recursos modernos, compatível com Ryzen 7000. *Preço:* \~R\$ 1.400.
* **RAM:** 32 GB (2×16 GB) DDR5 5200MHz Corsair Vengeance (ou similar). *Preço:* \~R\$ 700.
* **Armazenamento:** SSD NVMe M.2 1 TB (PCIe 4.0, ex: Kingston NV2 ou Samsung 980). *Preço:* \~R\$ 500.
* **GPU:** **Vídeo integrado** (AMD Radeon RDNA2 onboard do Ryzen) – dispensa GPU dedicada.
* **PSU:** Fonte 750W 80 Plus Gold semi-modular (ex: Corsair RM750). *Preço:* \~R\$ 600. (Sobra margem para upgrades futuros, alta eficiência).
* **Gabinete:** Mid-tower ATX (ex: Cooler Master TD500 ou similar) com bom airflow. *Preço:* \~R\$ 400.

*Total estimado:* **\~R\$ 8.500–9.000**. Esta configuração foca no melhor CPU e mantém alto equilíbrio nos demais componentes. Com 16 núcleos e SSD rápido, não haverá gargalos de CPU ou I/O. Há ainda margem no orçamento para ampliar RAM para 64 GB no futuro, se necessário.

### Build 2 – **Desktop AMD Custo-Benefício (Eficiência e Equilíbrio)**

**Descrição:** Montagem visando **excelente performance por real investido**. Usa um CPU ligeiramente inferior ao Build 1, mas ainda extremamente potente, economizando em pontos que não afetam o uso pretendido.

* **CPU:** AMD Ryzen 9 7900 (12 núcleos / 24 threads, até 5.4GHz) – Versão 65W eficiente. Inclui cooler Wraith Prism no pacote. *Preço:* \~R\$ 2.650. Apesar do TDP menor, entrega desempenho muito próximo ao 7900X, com vantagem de vir com cooler e consumir menos energia (mais silencioso).
* **Cooler CPU:** **(Opcional)** Usar cooler box Wraith Prism incluso. Pode-se manter ele inicialmente (suporta o 7900 a 65W). Se quiser ainda menos ruído, adicionar depois um cooler torre \~R\$250 (ex: DeepCool Gammaxx, Hyper 212).
* **Placa-mãe:** ASRock B650 PG Riptide ou Gigabyte B650 Aorus (AM5, DDR5). *Preço:* \~R\$ 1.000–1.200 (um pouco mais simples que a Asus TUF, porém suficiente para o Ryzen 7900 de 65W).
* **RAM:** 32 GB (2×16 GB) DDR5 5200MHz. *Preço:* \~R\$ 700. (Mesma especificação do Build 1).
* **Armazenamento:** SSD NVMe 1 TB PCIe 4.0 (ex: Western Digital SN770). *Preço:* \~R\$ 500.
* **GPU:** **Vídeo integrado** (Radeon do Ryzen 7900).
* **PSU:** Fonte 650W 80 Plus Gold (ex: XFX XPG Core 650G). *Preço:* \~R\$ 500. (650W cobre folgado, dado o uso de iGPU e CPU 65W).
* **Gabinete:** Mid-tower mATX/ATX básico (ex: Aerocool, Redragon) com 2 fans. *Preço:* \~R\$ 300.

*Total estimado:* **\~R\$ 6.500–7.000**. Este build economiza \~R\$2.000 vs. o Build 1, principalmente pelo CPU mais barato e ausência de cooler extra, sem sacrificar capacidade para o caso de uso. Os 12 cores dão conta dos vários processos em paralelo. O dinheiro economizado pode, por exemplo, ser investido em **dobrar a RAM para 64 GB** (caso futuramente necessário) e ainda permanecer dentro do teto de R\$10k.

### Build 3 – **Workstation Pronta Dell Precision (32GB RAM)**

**Descrição:** Solução de **workstation de marca**, já montada e testada de fábrica, com suporte e garantia on-site. Ideal para quem prefere um sistema pronto, sem necessidade de montar manualmente. A escolha aqui é uma **Dell Precision 3660 Tower** configurada para nosso uso.

* **CPU:** Intel Core i7-13700 (16 núcleos/24 threads totais – arquitetura 8P+8E cores, 5.2GHz turbo) – Processador robusto de 13ª geração, excelente em single-thread e multi-thread.
* **RAM:** 32 GB DDR5 4400MHz (geralmente em 2×16 GB nos Precision). Expansível até 128 GB.
* **Armazenamento:** SSD NVMe M.2 1 TB (Classe 40 Dell, equivalentes a PCIe 4.0 x4) para sistema e dados.
* **GPU:** NVIDIA T1000 4 GB (GPU profissional de entrada, arquitetura Turing) – suporta até quatro monitores, suficiente para uso 2D e decodificação de vídeo. *Obs.:* Pode ser útil instalar o driver proprietário NVIDIA no Linux para melhor suporte gráfico.
* **Outros:** Fonte 500W 80 Plus Platinum (das Precision), gabinete torre com refrigeração otimizada (opção de coolers avançados da Dell), diversas portas USB, etc.

Essa configuração é próxima de uma ofertada no mercado: *Precision 3660 com i7-13700, 32GB, SSD 2TB, T1000*, encontrada por **\~R\$ 9.780**. A máquina vem normalmente com Windows 11 Pro, mas suporta completamente Linux (a Dell inclusive vende algumas configurações com Ubuntu). Seria questão de instalar a distribuição Linux de preferência após a compra.

*Total estimado:* **\~R\$ 9.500–10.000**. Fica no limite do orçamento, mas já inclui todos os componentes e garantia. A vantagem aqui é a praticidade (sem montagem manual) e recursos empresariais (TPM, chassis otimizado, suporte Dell). A desvantagem é menos *flexibilidade de upgrades* – porém, ainda é possível adicionar mais RAM ou trocar o SSD facilmente. Esta workstation pronta entrega desempenho equivalente aos desktops acima, com qualidade assegurada de construção.

## Justificativas Técnicas Finais

As configurações propostas evitam gargalos ao equilibrar CPU multinúcleo, memória farta e armazenamento veloz. Escolhemos componentes essenciais de alta performance – **processador poderoso, RAM abundante, SSD NVMe** – conforme as recomendações de especialistas para cargas de IA e banco de dados (maior é melhor nesses quesitos). Ao mesmo tempo, reduzimos custos em áreas de menor impacto (como GPU dedicada e luxo de hardware) para manter tudo dentro do orçamento.

Essa máquina resultante será capaz de **rodar múltiplos processos Node.js simultaneamente** sem saturar CPU (graças aos vários núcleos), **consultar bases SQL locais rapidamente** (aproveitando operações paralelas do PostgreSQL e caching em RAM) e lidar com fluxos de trabalho intensos de desenvolvimento (várias IDEs, containers, etc.) sem lentidão. Com Linux otimizado, o desenvolvedor terá um ambiente responsivo mesmo sob cargas complexas, atendendo às exigências do projeto AGI proposto.
