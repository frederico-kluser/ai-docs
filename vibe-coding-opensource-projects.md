# Projetos Open-Source para *VIBE Coding* (Agentes de Código com LLM)

**VIBE Coding** refere-se a uma forma de programar orientada por IA, em que o desenvolvedor guia um agente de *Large Language Model* (LLM) que gera e modifica o código. Ferramentas como Cursor IDE, GitHub Copilot (Agent mode) e Windsurf IDE exemplificam essa abordagem, oferecendo assistentes de código autônomos integrados ao fluxo de desenvolvimento. A seguir, apresentamos os melhores projetos open-source focados exclusivamente nesse estilo de codificação *“vibe”*, que replicam funcionalidades semelhantes, atendendo aos critérios de: código totalmente aberto e modificável, uso via terminal, suporte a múltiplas instâncias paralelas, e integração com Git, terminal e buscas web para auxiliar na codificação.

## Detalhes dos Projetos

### **Aider** – Assistente de Programação em Parelha no Terminal

**Aider** é um agente de codificação AI projetado para uso direto no terminal, focado em integração profunda com Git. Criado por Paul Gauthier, ele permite *pair programming* com modelos LLM como Claude ou GPT, realizando modificações de código diretamente no repositório e fazendo *commits* automáticos com mensagens descritivas. Ao contrário de simples completadores de código, o Aider pode lidar com múltiplos arquivos simultaneamente e manter contexto de todo o projeto, o que viabiliza refatorações complexas. Ele suporta modelos locais ou em nuvem de forma flexível e aberta a customizações (licença Apache-2.0). Além de sugerir e editar código via chat, o Aider consegue executar testes e linters a cada alteração que faz, corrigindo problemas encontrados de forma iterativa. Com mais de 8 mil estrelas no GitHub e uma comunidade ativa, é considerado um dos assistentes de código abertos mais avançados atualmente, frequentemente mencionado como referência em relação a ferramentas fechadas. *Repositório:* **[Aider-AI/aider](https://github.com/Aider-AI/aider)** (Python, CLI).

### **Plandex** – Agente Autônomo para Grandes Tarefas de Código

**Plandex** é um agente de desenvolvimento de código autônomo, operado via terminal, projetado para planejar e executar tarefas de programação de múltiplos passos em projetos grandes. Ele se destaca por suportar contextos extremamente extensos (até \~2 milhões de tokens, indexando diretórios com 20+ milhões de tokens usando mapas de projeto via *tree-sitter*) para compreender e modificar dezenas de arquivos de uma vez. O Plandex combina modelos de diferentes provedores (OpenAI, Anthropic, Google, etc.) para melhorar resultados e pode funcionar de modo totalmente autônomo – carregando arquivos relevantes, elaborando um plano, implementando mudanças, executando comandos e até depurando erros sem intervenção humana. Todas as alterações geradas ficam em um sandbox de diffs cumulativos, separados do código real até serem aprovados, facilitando revisão e revertendo se necessário. Ele possui integração opcional com Git: inicializa repositórios e pode gerar *commits* automaticamente com mensagens contextuais. Embora não realize buscas web, o Plandex foca em robustez para projetos reais de grande porte, sendo resiliente em bases de código extensas onde outras ferramentas falham ou alucinam mudanças incorretas. *Repositório:* **[plandex-ai/plandex](https://github.com/plandex-ai/plandex)** (Go/Python, CLI).

### **Continue** – Extensão de IDE Personalizável para Assistência de Código

**Continue** é uma plataforma open-source que fornece assistentes de código inteligentes via extensões para IDEs (VS Code, JetBrains). Ao instalar a extensão, o desenvolvedor ganha recursos similares ao GitHub Copilot Chat dentro do editor: completa código automaticamente enquanto você digita (suporte multilínguas), permite selecionar trechos e dar instruções em linguagem natural para refatorar ou gerar código, e oferece um chat onde é possível fazer perguntas sobre funções, arquivos ou o projeto inteiro, com o contexto do código fornecido. O Continue destaca-se por sua **hub** configurável de modelos e regras – é possível conectar qualquer modelo (OpenAI, Ollama para modelos locais, etc.) e personalizar o comportamento do assistente através de *prompts* ou persona específicas. Todo o código do core e extensões é aberto (licença Apache-2.0) e a comunidade contribui com *plugins* e melhorias constantemente. Com mais de 25 mil estrelas no GitHub, é um dos projetos mais populares da categoria, indicando forte atividade e suporte comunitário. No entanto, a ferramenta em si opera dentro do editor gráfico (não via terminal), e não possui agentes executando comandos de terminal ou buscas web integradas por padrão – seu foco está em completar e modificar código com contexto, de forma altamente customizável e **self-hosted**. *Repositório:* **[continuedev/continue](https://github.com/continuedev/continue)** (TypeScript, Extensão VS Code/JetBrains).

### **Void** – IDE Autônoma Open-Source (Fork do VS Code)

**Void** é apresentado como a alternativa open-source ao Cursor IDE. Trata-se de um editor completo derivado do código-fonte do Visual Studio Code, com integração nativa de recursos de IA similares aos do Cursor. Desenvolvido pela Void Editor, ele traz todos os recursos centrais do Cursor em um ambiente totalmente personalizável. O objetivo do Void é permitir que o agente de IA tenha um entendimento maior do seu projeto local – incluindo habilidade de editar múltiplos arquivos em conjunto e navegar pelo sistema de arquivos – sem depender de serviços fechados. Por ser um fork do VSCode, ele mantém compatibilidade com extensões e a interface familiar, incluindo terminal embutido e controle de versão Git integrados à IDE. Toda a lógica de *prompts* e interações do agente roda localmente, podendo conectar-se a modelos LLM à escolha do usuário (OpenAI, Anthropic, etc.) diretamente, mantendo os dados privados. Em termos de recursos atuais, o Void ainda está em estágio beta (cerca de 1 ano de desenvolvimento ativo) e evoluindo rapidamente – já suporta assistente de código via chat, geração de trechos e explicações, mas ainda expande funcionalidades. Não há indicação de que execute buscas na web ou comandos autônomos, focando por enquanto em edição inteligente de código no próprio editor. O projeto é ativo e atrai contribuições (12k+ stars). *Repositório:* **[voideditor/void](https://github.com/voideditor/void)** (TypeScript, Aplicação de Desktop).

### **Kilo Code (Cline/Roo)** – Plugin VS Code *Agentic* com Uso de Ferramentas Externas

**Kilo Code** é um plugin open-source para VS Code que nasceu da fusão de dois projetos da comunidade, **Cline** e **Roo**, conhecidos por prover uma experiência de “Cursor gratuito” dentro do VSCode. O Kilo se descreve como um *“superset”* dessas ferramentas, incorporando todas as suas funcionalidades e adicionando outras. Diferentemente de completadores simples, o Kilo/Cline atua de forma **agente**, capaz de executar uma série de passos para cumprir uma tarefa de codificação: ele pode gerar código, avaliar o resultado (por exemplo, compilando ou rodando testes via terminal integrado), identificar e corrigir erros por conta própria, e então continuar o processo até concluir a tarefa proposta. Na prática, o agente pode usar várias *tools*: por exemplo, consegue rodar comandos de terminal no ambiente de desenvolvimento para validar sua saída ou checar se os testes passam, e até mesmo abrir um navegador embutido para inspecionar páginas web locais geradas pelo código (como a interface de uma aplicação em execução). Também há integração com serviços de busca: o Kilo inclui conectores para ferramentas como o **Perplexity AI** (buscador AI) e pode utilizá-los para obter contexto externo ou documentação quando necessário. Possui integração com Git e outras utilidades de desenvolvimento – de acordo com seus autores, ele vem com *“baterias inclusas”*, suportando uso de Git, ferramentas de UI (ex: Figma via Magic UI) e outros recursos out-of-the-box. O Kilo Code é totalmente open-source (o plugin VSCode sob licença MIT) e se financia via serviços empresariais, mantendo um ritmo de desenvolvimento ativo em sintonia com as atualizações do Cline e Roo. É uma opção promissora e bastante completa para quem busca um “Copilot Agent” livre dentro do VS Code. *Repositório:* **[kilocodeai/kilo](https://github.com/kilocodeai/kilo)** (TypeScript, Extensão VS Code). *(Cline e Roo, que deram origem ao Kilo, também estão disponíveis: ex. **[Cline](https://github.com/Unlock-Creativity/Cline)**.)*

### **Aide (CodeStory)** – IDE AI-Nativa (fork do VS Code) com PRs Automatizados

**Aide** foi uma IDE open-source com foco em AI, desenvolvida pela startup CodeStory (YC S23), que buscava ser *“a primeira IDE nativa em IA”*. Baseada no VS Code, ela incorporava um agente inteligente de forma transparente no editor. Com Aide, podia-se conversar com o agente sobre o código (chat integrado ao editor) e aplicar edições sugeridas diretamente nos arquivos, inclusive múltiplos arquivos de uma vez, mantendo o desenvolvedor no fluxo. Um recurso marcante era a atuação proativa do agente: por exemplo, após gerar código, ele automaticamente checava erros de lint ou compilação via linguagem de servidor (LSP) e já sugeria correções, ou puxava definições e referências relevantes no projeto para enriquecer o contexto da resposta. O Aide tinha integração nativa com GitHub – orientado a fluxos de *Pull Request*: permitia criar tarefas em segundos e abrir PRs automatizados com as alterações propostas, que podiam ser iteradas pelo agente (incluindo revisão de código e sugestões de melhoria em ciclos contínuos). Um diferencial era o **suporte à paralelização**, permitindo rodar **vários agentes em paralelo** para dividir tarefas e acelerar a implementação de funcionalidades complexas. Apesar da recepção inicial positiva (2.1k stars no GitHub) e de atingir o topo do benchmark *swebench-lite* na época, o projeto Aide foi descontinuado pelos autores após alguns meses (o repositório foi arquivado em 2024). A CodeStory direcionou seus esforços para uma solução fechada chamada *AgentFarm*, mas o código-fonte do Aide permanece disponível para a comunidade que queira estudá-lo ou criar *forks*. *Repositório:* **[codestoryai/aide](https://github.com/codestoryai/aide)** (TypeScript, Desktop IDE – **Projeto Arquivado**).

### **GPT-Engineer** – Geração de Código Completo a partir de Especificações

**GPT-Engineer** é uma ferramenta CLI open-source que exemplifica o conceito de vibe coding extremo: você descreve em linguagem natural o software desejado, e a IA se encarrega de gerar uma base de código funcional seguindo essas instruções. Ele foi um dos primeiros projetos populares a experimentar automação total de programação usando GPT-4. O fluxo típico envolve criar um prompt com requisitos do projeto; a IA pode interagir fazendo perguntas de esclarecimento e então produz o código completo. O GPT-Engineer não apenas escreve os arquivos, mas também *executa o código gerado* para testar ou rodar a aplicação quando aplicável, avaliando o resultado e possibilitando ciclos de melhoria adicionais. Dessa forma, ele consegue detectar erros de execução e ajustar o código em seguida, num loop até atender à especificação (com alguma supervisão do usuário para refinamento). Embora não tenha integração nativa com Git ou acesso à web, o GPT-Engineer se foca em orquestrar chamadas de LLM para construção de projetos do zero, servindo como plataforma de experimentação em geração de código. Seu código é totalmente aberto (MIT) e extensível; porém, o mantenedor original passou a evoluir a ideia em um serviço proprietário (*lovable.dev*), e recomenda-se hoje usar o Aider para um CLI mais “hackável” e mantido ativamente. Ainda assim, o GPT-Engineer influenciou muitas outras ferramentas e permanece como referência para estudos de agentes autônomos de codificação. *Repositório:* **[AntonOsika/gpt-engineer](https://github.com/AntonOsika/gpt-engineer)** (Python, CLI).

**▶ Observação Final:** Todos os projetos acima são open-source e permitem modificações livres em seu código. Ao escolher uma ferramenta, considere o nível de atividade da comunidade: projetos como **Aider**, **Continue**, **Plandex** e **Kilo/Cline** demonstram forte manutenção e evolução constante, enquanto o **Aide** foi descontinuado e o **GPT-Engineer** tem desenvolvimento mais lento recentemente. Cada um oferece abordagens ligeiramente diferentes para incorporar LLMs no fluxo de desenvolvimento – desde extensões leves até IDEs completas ou ferramentas de terminal – cabendo ao desenvolvedor avaliar qual se integra melhor ao seu caso de uso e preferência de workflow.

# Ferramentas Open-Source

* **Aider** – É um assistente de programação em terminal. Ele *mapeia todo o código do repositório* (facilitando projetos grandes) e conecta-se a LLMs locais ou na nuvem (Claude 3.7 Sonnet, GPT-4o etc.). Com alto desempenho (atingiu das maiores pontuações no benchmark SWE-Bench), oferece sugestões de código de alta qualidade. Suporta edições multiarquivos em cascata (faz *diffs* automáticos) e integrações Git automáticas (commits sugeridos com mensagens razoáveis). Tem ainda recursos de voz e de contexto visual (pode usar imagens/webpages como contexto). Por ser CLI, tem curva de uso maior (menos amigável que um IDE), mas quem gosta de terminal elogia sua robustez e familiaridade. A comunidade é ativa (várias issues no GitHub e discussões), respaldando seu desenvolvimento rápido.

* **Plandex** – Agente open-source focado em tarefas muito grandes. A versão atual lida com janelas de contexto de \~2 milhões de tokens (e indexa 20M em projetos de escala). Usa vários modelos (OpenAI, Gemini, Claude, Anthropic etc.) em conjunto para maior confiabilidade. Possui modos autônomos (planejamento, execução de comandos, compilação, teste e até *auto-debug*). Além disso, tem um sistema de revisão em sandbox: as mudanças propostas são aplicadas num *branch* virtual até serem aprovadas, evitando bagunça no código final. Essas capacidades dão alta qualidade e versatilidade nas sugestões, mas a configuração é mais complexa. A estabilidade vem melhorando (v2 focou em confiabilidade) e a comunidade vem crescendo (GitHub ativo, Discord para contribuições). Em resumo, Plandex oferece *altíssima* capacidade multi-arquivo e automação, ideal para grandes bases de código, mas exige recursos (CPU/GPU) e ainda está em polimento de UI e bugs.

* **Continue** – Extensão open-source para VS Code e JetBrains. Permite “*conversar com todo seu código*”: via sidebar de chat você faz perguntas ao código, pede refatorações, explicações ou gerações de trechos, tudo via prompts naturais. Suporta modelos locais ou remotos (GPT-4, Claude, Mistral etc.), evitando bloqueio em um único provedor. A usabilidade é alta por ser integrada na IDE, com interface intuitiva de chat. As sugestões de código são boas (dependem do modelo usado) e edita múltiplos arquivos de forma transparente. Por ser jovem, pode ter limitações em projetos muito grandes (indexação inicial lenta) ou pequenos bugs menores, mas é elogiada por sua flexibilidade (extensões e configurações via Python e prompts). A comunidade está em crescimento (mais ativa em GitHub e fórum), mas menor que ferramentas de grandes empresas.

* **Void Editor** – Open-source considerado *“Cursor alternativo”*. É um editor de código baseado em Electron (semelhante ao VSCode) que se conecta a qualquer LLM, inclusive auto-hospedados, garantindo privacidade total dos dados. Entre os recursos chave estão: consciência de sistema de arquivos (responde perguntas sobre todo o projeto via índice de arquivos) e **transparência total** (o usuário pode visualizar e editar internamente todos os prompts enviados ao modelo). Possui aplicação super-rápida de mudanças mesmo em arquivos gigantes (faz o mínimo necessário em cada trecho). Ainda em beta aberta (lançado via Y Combinator em 2024), estável mas menor, traz solução robusta de multi-arquivo e suporta modelos customizados pelo usuário. O foco é privacidade e comunitário – sendo open-source, qualquer um pode sugerir extensões. A atividade da comunidade é moderada (crowd ainda pequeno), mas é muito promissor em termos de flexibilidade e desempenho.

* **Kilo Code (Cline/Roo)** – Conjunto de extensões VS Code open-source. Kilo Code é o superset das extensões Cline e Roo, oferecendo *múltiplos modos*: **Architect** (planejamento), **Coder** (codificação) e **Debugger** (debug). Entre os recursos: gera código a partir de linguagem natural, refatora automaticamente, executa comandos de terminal e até “automação de navegador” (interação web). Verifica o próprio código (checando testes e linter). Possui um marketplace (MCP Servers) para adicionar funcionalidades via servidores de contexto. Vem pronto para usar com credenciais de GPT-4.1, Claude 3.7, Gemini 2.5 etc. (dá créditos grátis para testes). Em geral, fornece alta flexibilidade e qualidade nas sugestões, similar a ferramentas comerciais, porém a interface ainda pode ser complicada e depende de um ambiente VSCode limpo. A comunidade (GitHub e Discord) é ativa, com atualizações semanais.

* **Aide** – Editor open-source baseado em VS Code (foi um fork dedicado). Integra chat e edição no próprio editor: você conversa (@arquivos) e o assistente edita o código em múltiplos arquivos automaticamente. Traz recursos avançados como rollback nativo (desfaz em um clique todas as mudanças feitas pela AI) e widget inline (Cmd+K) para instruções rápidas. A manipulação do LLM é feita por um serviço local (“sidecar”), garantindo privacidade. A qualidade das sugestões é alta (modelos top são usados) e o IDE é projetado para cooperar com o dev. Como pontos fracos, manter um fork próprio do VSCode gera *bugs de manutenção* frequentes, como observado na comunidade. Assim, embora potente (qualidade alta, multiarquivo, UX rica), a estabilidade e extensibilidade podem ser inferiores a plugins de VSCode oficiais. A atividade da comunidade é moderada (poucas contribuições externas até agora).

* **GPT-Engineer** – Plataforma CLI open-source para geração automatizada de projetos inteiros. O usuário descreve o projeto em linguagem natural e o sistema engaja num diálogo de refinamento, depois monta o código (geralmente uma aplicação web completa) automaticamente. É orientado a desenvolvedores (não tanto novatos) e funciona muito bem para projetos web com stack conhecido. Suporta múltiplos modelos (da OpenAI ou abertos) e permite configurar “identidade” do agente, mas exige que o usuário revise, teste e integre as partes geradas. É gratuito (MIT) e tem grande comunidade (54k ⭐ no GitHub). Um ponto negativo é a especialização: foca em projetos Web-App (não gera, por exemplo, apps mobile *por enquanto*). Recentemente foi lançado um App comercial (gptengineer.app) para apoiar o projeto open-source. Em suma, o GPT-Engineer acelera prototipagem de projetos reais, mas não substitui completamente a supervisão humana (exige bons prompts) e sua atividade comunitária é alta graças ao modelo aberto.

# Ferramentas Comerciais

* **Windsurf IDE (Codeium)** – Editor proprietário “agentivo” lançado pelo Codeium. Integra o agente *Cascade*, que “segue 10 passos à frente” do usuário, propondo correções antes de erros acontecerem. Oferece completions muito avançadas e histórico de comandos (*Windsurf Tab*) para contexto de fluxo de trabalho. Na prática, usuários elogiam a qualidade das sugestões (baseadas em modelos avançados de IA), mas notaram limitações: o modelo às vezes erra contextos de projeto (por ex. não percebeu uso de Tailwind até ser instruído). Pode rodar builds e analisar erros via Chrome devtools embutido, mas muitos preferem depurar externamente. Em termos de UX, tem interface moderna e integração completa (preview e deploy embutido), mantendo o fluxo dentro do IDE. Como desvantagem, é pago: houve reclamações da comunidade (no Reddit) sobre táticas de cobrança agressivas em preço de assinatura. Em resumo, Windsurf entrega qualidade MUITO ALTA de sugestões e fluxo unificado, mas é fechado, caro, e ainda amadurece em relação à política comercial.

* **Cursor IDE** – IDE comercial baseado em VSCode. Coloca a IA como núcleo da edição: fornece autocompletar conscientes do projeto todo, refatoração e depuração inline por comandos naturais. Também explica trechos de código e propõe melhorias baseadas no cursor. Destina-se a acelerar iteração e onboarding em código complexo. A experiência de uso é refinada (interface elegante similar a VSCode) e as sugestões são de alta qualidade (usa GPT-4o, Gemini, etc.). É também ferramenta fechada e paga, com suporte oficial, então é estável. Não permite extensões externas de IA (ao contrário do open-source), mas possui suporte corporativo. A comunidade de usuários é pequena (produto novo), mas inclui desenvolvedores individuais e algumas equipes.

* **GitHub Copilot Agents** – Produto da GitHub/Microsoft integrado ao VSCode. Além de completar código em linha, sua *Agent Mode* (lançado em 2025) torna o Copilot capaz de executar tarefas completas: gerar código, detectar e corrigir seus próprios erros, sugerir/rodar comandos de terminal e iterar até completar o pedido inicial. Suporta edição multi-arquivo (o blog fala explicitamente em *“multi-file edits”*) e modelos avançados (inclui GPT-4o e Gemini 2.0 Flash). Como vantagem, é extremamente integrado a repositórios GitHub (geração de pull requests automática, reviews AI, etc.) e tem grande base de usuários. Em qualidade de sugestão e estabilidade (infra Microsoft/Azure) é muito alta. Desvantagens: é fechado/fechado por assinatura (embora haja free para projetos OSS) e pode apresentar sugestões erradas (como todo AI), mas permite iteração rápida. A comunidade é massiva, dado que Copilot é largamente adotado no mercado.

* **Google Cloud Code (Gemini Code Assist)** – Solução de AI da Google para desenvolvimento. Inclui o **Gemini Code Assist** em IDEs populares (VSCode, JetBrains, Cloud Shell). Ele completa código à medida que escreve e pode gerar trechos inteiros a partir de comentários. Também fornece chat integrado para tirar dúvidas de programação e orientação de boas práticas. Dispõe de “agentes” que auxiliam em tarefas (geração de testes, migração de código, análise de PRs, documentação, etc.). Um diferencial é o uso de todo o código local como contexto: pode fazer transformações em larga escala (atualização de dependências cruzadas, refactoring múltiplos arquivos, revisão inteligente). Usa o modelo Gemini 2.5 (altíssima performance em code-gen). Em termos de UX e estabilidade é profissional (monitoramento, dashboard de métricas, logs). Por ser empresa Google, é mais voltado a clientes corporativos e projetos na nuvem Google. Suas limitações são: é fechado, e melhor aproveitado apenas dentro do ecossistema Google Cloud. Ainda assim, pelas capacidades oferecidas, entrega uma experiência de **alta qualidade** para quem trabalha em ambiente Google, com operação sólida e segurança empresarial.
